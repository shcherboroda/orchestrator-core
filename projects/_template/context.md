# Project Context Pack (Template)

## 1. What this project is
- One-paragraph description.

## 2. Current state
- What exists today (repos, services, deployed pieces).
- What is confirmed working.

## 3. Target state (Definition of Done)
- Bullet list of user-visible flows and technical requirements.

## 4. Architecture (high-level)
- Components and responsibilities.
- Key design decisions.

## 5. Code locations
- Repo paths and important folders.

## 6. Known gaps / broken flows
- What is missing / failing right now.

## 7. Constraints
- Budget, infra, time, security, modularity, etc.

## 8. Acceptance criteria
- Concrete verifiable conditions for v0.
